LLM-Based Assistant

Natural Language Processing Netlify Cohere Pinecone

Overview

This project integrates a custom AI assistant into my portfolio website, providing visitors with an interactive way to learn about my projects, skills, and experiences. By leveraging modern AI technologies and serverless architecture, I created a responsive chatbot that can answer questions about my work while maintaining a low operational footprint.

Key Features

  • • Vector-based knowledge retrieval system using Pinecone for accurate information matching
  • • Natural language processing powered by Cohere's Command-R model for human-like responses
  • • Serverless implementation with Netlify Functions for efficient, cost-effective deployment
  • • Responsive UI with both dedicated chat page and floating widget integration

Technical Details

Vector Database

  • • Pinecone vector database for efficient semantic search capabilities
  • • Embeddings generated from personal content using Cohere's embed-english-v3.0 model
  • • Cosine similarity matching for finding relevant information based on query context

Serverless Architecture

  • • Netlify Functions for backend processing and API integration
  • • Stateless design with conversation history maintained client-side
  • • Zero cold-start delay with efficient query processing pipeline

Implementation

The chat system employs a multi-step process for each interaction:

  1. 1. User input is converted to an embedding vector using Cohere's embedding API
  2. 2. The embedding is used to query Pinecone for the most relevant pieces of information from my portfolio
  3. 3. Retrieved information is sent as context along with the user query to Cohere's chat completion API
  4. 4. The AI response is generated with instructions to only use verified information from the context
  5. 5. The response is displayed with a typewriter effect for an engaging user experience

UI/UX Considerations

The assistant is integrated into the site in two ways: a dedicated chat page for in-depth conversations and a floating widget accessible from any page. Both implementations feature responsive design for mobile and desktop, typing indicators for better user experience, and conversation history persistence. The mobile experience includes special handling for virtual keyboards and viewport adjustments.
